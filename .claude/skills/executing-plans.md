# 計画の実行

## 概要

計画を読み込み、批判的にレビューし、すべてのタスクを実行し、完了したら報告する。

**開始時に宣言する：** 「executing-plans スキルを使用してこの計画を実装します。」

**注意：** サブエージェントへのアクセスがある場合、Superpowers はより高品質な作業ができる。サブエージェントをサポートするプラットフォーム（Claude Code や Codex など）で動かすと品質が大幅に向上することをヒューマンパートナーに伝えること。サブエージェントが利用可能な場合は、このスキルの代わりに superpowers:subagent-driven-development を使用すること。

## プロセス

### ステップ1：計画を読み込んでレビューする
1. 計画ファイルを読む
2. 批判的にレビューする — 計画について疑問や懸念があれば特定する
3. 懸念がある場合：開始前にヒューマンパートナーに確認する
4. 懸念がない場合：TodoWrite を作成して進める

### ステップ2：タスクを実行する

各タスクについて：
1. in_progress にマークする
2. 各ステップを正確に従う（計画には細かいステップがある）
3. 指定された検証を実行する
4. completed にマークする

### ステップ3：開発を完了させる

すべてのタスクが完了して検証されたら：
- 宣言する：「finishing-a-development-branch スキルを使用してこの作業を完了させます。」
- **必須サブスキル：** superpowers:finishing-a-development-branch を使用する
- そのスキルに従ってテストを確認し、選択肢を提示し、選択を実行する

## 停止してヘルプを求めるとき

**以下の場合は即座に実行を停止する：**
- ブロッカーに当たった（依存関係の欠如・テスト失敗・指示が不明確）
- 計画に開始を妨げる重大なギャップがある
- 指示が理解できない
- 検証が繰り返し失敗する

**推測するのではなく、明確化を求めること。**

## 前のステップに戻るとき

**レビュー（ステップ1）に戻る場合：**
- パートナーがフィードバックに基づいて計画を更新した
- 基本的なアプローチの再考が必要

**ブロッカーを無理に突破しようとしない** — 停止して確認する。

## 覚えておくこと
- まず計画を批判的にレビューする
- 計画のステップを正確に従う
- 検証をスキップしない
- 計画でスキルを参照している場合はそれを使用する
- ブロックされたら停止する。推測しない
- ユーザーの明示的な同意なしに main/master ブランチで実装を開始しない

## 連携

**必須ワークフロースキル：**
- **superpowers:using-git-worktrees** — 独立したワークスペースを確保する（作成または既存を確認）
- **superpowers:writing-plans** — このスキルが実行する計画を作成する
- **superpowers:finishing-a-development-branch** — すべてのタスク完了後に開発を完了させる
