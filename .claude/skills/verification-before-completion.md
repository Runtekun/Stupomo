# 完了前の検証

## 概要

検証なしに作業が完了したと主張することは、効率ではなく不誠実だ。

**コア原則：** 常にエビデンスが先、主張は後。

**このルールの文言に違反することは、ルールの精神に違反することだ。**

## 鉄則

```
新鮮な検証エビデンスなしに完了の主張をしてはならない
```

このメッセージで検証コマンドを実行していなければ、通過したと主張することはできない。

## ゲート関数

```
任意のステータスを主張したり満足感を表明したりする前に：

1. 特定する：何のコマンドがこの主張を証明するか？
2. 実行する：完全なコマンドを実行する（新鮮な、完全な）
3. 読む：完全な出力、終了コードを確認する、失敗を数える
4. 確認する：出力は主張を確認しているか？
   - いいえの場合：エビデンスと共に実際のステータスを述べる
   - はいの場合：エビデンスと共に主張を述べる
5. その後のみ：主張を行う

いずれかのステップをスキップ = 確認ではなく嘘
```

## よくある失敗

| 主張 | 必要なもの | 不十分なもの |
|------|----------|------------|
| テストが通過する | テストコマンドの出力：0件の失敗 | 以前の実行、「通過するはず」 |
| リンターがクリーン | リンターの出力：0件のエラー | 部分的なチェック、外挿 |
| ビルドが成功する | ビルドコマンド：終了コード0 | リンター通過、ログが良さそう |
| バグが修正された | 元の症状をテスト：通過 | コードを変更した、修正されたと仮定 |
| リグレッションテストが動く | レッド・グリーンサイクルを確認 | テストが一度通過 |
| エージェントが完了した | VCSの差分が変更を示す | エージェントが「成功」を報告 |
| 要件が満たされた | 行ごとのチェックリスト | テストが通過 |

## 要注意サイン — 停止

- 「するはず」「おそらく」「そうらしい」を使っている
- 検証前に満足感を表明している（「素晴らしい！」「完璧！」「完了！」など）
- 検証なしにコミット/プッシュ/PRをしようとしている
- エージェントの成功報告を信頼している
- 部分的な検証に頼っている
- 「今回だけ」と考えている
- 疲れていて作業を終わらせたい
- **検証を実行せずに成功を示唆する言い回し**

## 合理化の防止

| 言い訳 | 現実 |
|--------|------|
| 「今は動くはず」 | 検証を実行する |
| 「自信がある」 | 自信 ≠ エビデンス |
| 「今回だけ」 | 例外なし |
| 「リンターが通過した」 | リンター ≠ コンパイラ |
| 「エージェントが成功と言った」 | 独立して確認する |
| 「疲れている」 | 疲労 ≠ 言い訳 |
| 「部分的なチェックで十分」 | 部分的なものは何も証明しない |
| 「言葉が違うからルールは適用されない」 | 文言より精神を優先 |

## 主なパターン

**テスト：**
```
✅ [テストコマンドを実行] [表示：34/34 通過] 「すべてのテストが通過します」
❌ 「今は通過するはず」/ 「正しそう」
```

**リグレッションテスト（TDD レッド・グリーン）：**
```
✅ 書く → 実行（通過）→ 修正を元に戻す → 実行（失敗しなければならない）→ 復元 → 実行（通過）
❌ 「リグレッションテストを書いた」（レッド・グリーン検証なし）
```

**ビルド：**
```
✅ [ビルドを実行] [表示：終了コード0] 「ビルドが通過します」
❌ 「リンターが通過した」（リンターはコンパイルを確認しない）
```

**要件：**
```
✅ 計画を再読む → チェックリストを作成 → それぞれを確認 → ギャップまたは完了を報告
❌ 「テストが通過、フェーズ完了」
```

**エージェントの委譲：**
```
✅ エージェントが成功を報告 → VCSの差分を確認 → 変更を確認 → 実際の状態を報告
❌ エージェントの報告を信頼する
```

## なぜこれが重要か

24件の失敗記録から：
- ヒューマンパートナーが「信じない」と言った — 信頼が壊れた
- 未定義の関数が出荷された — クラッシュするだろう
- 欠落した要件が出荷された — 不完全な機能
- 偽の完了 → リダイレクト → 手戻りで時間が無駄に
- 違反：「誠実さは核心的な価値だ。嘘をつけば置き換えられる。」

## いつ適用するか

**以下の前に常に：**
- 成功/完了の主張のあらゆるバリエーション
- 満足感のあらゆる表明
- 作業状態についてのあらゆる肯定的な発言
- コミット・PR作成・タスク完了
- 次のタスクへの移行
- エージェントへの委譲

**ルールは以下に適用される：**
- 正確なフレーズ
- 言い換えと同義語
- 成功の含意
- 完了/正確さを示唆するあらゆるコミュニケーション

## まとめ

**検証に近道なし。**

コマンドを実行する。出力を読む。その後、結果を主張する。

これは交渉できない。
